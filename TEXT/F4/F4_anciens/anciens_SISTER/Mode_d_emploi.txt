
Pour ajouter les balises temporelles : utiliser SoundIndex. Après avoir utilisé SoundIndex, il faut ajouter automatiquement "xml:" devant toutes les traductions : les balises telles que
<TRANSL lang="en"
doivent être modifiés en : 
<TRANSL xml:lang="en"

Il faut également remplacer les chevrons ou leur expression régulière (&lt; &gt;) par un autre code pendant l'utilisation de SoundIndex sinon SoundIndex ne peut lire le document XML. Suggestion: CHEVRONGAUCHE et CHEVRONDROIT


Pour afficher un document en vue de sa vérification : utiliser les outils fournis par Séverine Guillaume :
-> showtext3.xsl est la feuille de style qui permet affichage et écoute du
son
-> qtPlayerManager.js est un petit programme qui permet l'écoute phrase à
phrase du son en fonction du découpage fait avec soundindex
-> showhide.js est un petit programme pour permettre d'afficher ou cacher
les traductions, transcriptions et gloses faites avec ite
-> play.gif et stop.gif sont juste des icones de lecture et arrêt du son

Pour exemple, il y a aussi un fichier xml d'annotations appelé
FOURMI.xml et le fichier audio FOURMI.wav qui va avec.

Mode de fonctionnement :

1- Mettre ton fichier xml d'annotations ainsi que ton fichier audio wavdans ce dossier.


Ensuite il faut ajouter 2 lignes dans ton fichier xml comme ceci :

2- Ajouter <?xml-stylesheet type="text/xsl" href="showText3.xsl"?> juste
sous la 1ère ligne du fichier (<?xml version="1.0" encoding="utf-8"?>). 
Ceci permet de faire le lien entre le fichier xml et la feuille de style.

3- Ajouter <url_sound>nom_du_fichier_audio.wav</url_sound> juste après la
balise <TEXT>. "nom_du_fichier_audio.wav" doit bien sûr être remplacé par
le nom de ton fichier audio.

Ceci donne avec l'exemple du fichier FOURMI.xml :

<?xml version="1.0" encoding="utf-8"?>
<?xml-stylesheet type="text/xsl" href="showText3.xsl"?>
<TEXT>
  <url_sound>FOURMI.wav</url_sound>
 ....(suite du fichier xml)

 
4- Ensuite, il suffit d'ouvrir le fichier xml dans un navigateur (firefox par
exemple) pour voir le texte interlinéaire de manière plus conviviale que le xml et d'écouter le fichier audio en même temps.
